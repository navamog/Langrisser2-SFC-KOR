# 데어랑그릿사 (Der Langrisser, SNES) 한글패치 — 초기 분석

## 대상 파일
| 파일 | 크기 | 설명 |
|------|------|------|
| `Der Langrisser (J) (V1.0).smc` | 2 MB | 일본어 원본 롬 (V1.0) |
| `Der Langrisser (J) (V1.1).smc` | 2 MB | 일본어 원본 롬 (V1.1) |
| `...[T+Eng100%_LangFanDood].smc` | 3 MB | 영문 100% 완역 롬 (LangFanDood) |
| `Current.ips` | 452 KB | 구버전 영문 부분번역 IPS (Moon Knight) — JP V1.0에 적용 시 3MB로 확장 |

## 롬 스펙
- 매퍼: **LoROM** (일본롬 mapmode 0x30 = FastLoROM)
- 일본롬 2MB → 영문판은 3MB로 **확장**(LoROM 최대 4MB이므로 약 1MB 여유 남음)
- 내부 헤더 제목(JP): 반각 가타카나 `デァランギリッサー`

## 텍스트 시스템 분석

### 텍스트는 평문 ASCII가 아님
"Langrisser" 문자열은 롬 헤더(0x7FC4)에만 존재. 게임 내 대사는 **커스텀 인코딩(테이블 방식)**.

### 일본어 롬 구조 (JP vs EN 2MB 구간 차이 분석)
- **0x110000 ~ 0x16FFFF (~360KB)**: 일본어 **한자/가나 폰트 타일** 영역.
  영문판이 이 구간을 영문 폰트/그래픽으로 통째 교체함.
- **0x000000 ~ 0x04FFFF**: 프로그램 코드 + 저지대 텍스트/포인터 (소량 변경).
- 일본어 대사는 **2바이트 코드**(한자 폰트를 인덱싱)로 저장된 것으로 추정.

### 영문판 구조
- **0x110000 ~ 0x16FFFF**: 영문 폰트(가변폭 8x16 추정) + 그래픽.
- **0x200000 ~ 0x2FFFFF (확장영역)**: 영문 번역 텍스트. `F7 xx` 이스케이프가
  반복되는 **사전(dictionary) 압축** 구조. 압축 때문에 직접 편집 난이도 높음.

## 전략적 시사점 (핵심)

한글은 완성형 2,350자(또는 조합형 자모) 글리프가 필요하며 **전각(16x16) 폰트**가 이상적.

- **일본어 롬을 기술적 베이스로 삼는 것이 유리**: 이미 2바이트 한자 폰트 엔진과
  ~360KB 글리프 영역이 존재 → 한자 글리프를 한글 글리프로 교체하고 텍스트 코드를
  리매핑하면 됨. CJK→CJK 팬번역의 정석 경로.
- **영문 롬을 베이스로 삼으면 불리**: 1바이트 가변폭 좁은 폰트 + 사전 압축과 싸워야 함.

→ **권장: 일본어 롬을 기술 베이스로, 영문판(롬/IPS)은 "대사 의미 참조용"으로 사용.**
  영문판은 각 대사가 무슨 뜻인지 즉시 알려주므로 일본어 해독 없이 번역 가능.

## 확보된 도구 (kor_patch/tools/)
- `ips.py` — IPS 파싱/적용/뱅크별 요약
- `romdiff.py` — 두 롬 바이너리 차이 분석
- `strings.py` — ASCII 문자열 추출
