# 텍스트 인코딩 체계 (해독·검증 완료)

emucap 라이브 디버거(Mesen 2.2.1)로 메시지 인터프리터를 디스어셈블하고,
실제 대사 스트림을 읽어 **정정·검증**했다.

## 메시지 인터프리터 (ROM, LoROM bank 0x00, 미러 0x80)
디스어셈블 요지 (`$66`=현재 코드, `$5D`=24bit 스트림 포인터, `$F4/$F5`=글리프 인덱스):
```
AD51 LDA $66 / CMP #$0A / BCS $AD5A / JMP $ADF0   ; <0x0A → 제어(점프테이블)
AD5A CMP #$F7 / BCC $AD91                          ; <0xF7 → 1바이트 글자
; 확장 인덱스(0xF7~0xFF):
AD5E SBC #$F7 / TAX                                ; X = code-0xF7 (0..8)
AD61 $F4=0xEC, $F5=0 ; loop X: $F5++,$F4--         ; → $F4/F5 = (0xEC-X)|(X<<8)
AD72 LDA [$5D] ; +$F4 → $F4/F5 ; INC $5D           ; 인자바이트 더해 최종 인덱스
; 1바이트 글자(0x0A~0xF6):
AD91 SBC #$0A → $F4 ; $F5=0                         ; 인덱스 = code-0x0A
AD98 JSR $8ABD / $B18E / $B3D8                      ; 글리프 렌더/업로드
```

## 인코딩 규칙 (확정)
| 코드 | 의미 |
|------|------|
| **0x00** | 메시지/페이지 종료 |
| **0x01~0x09** | 제어코드(점프테이블 @0xADF5). 관측: 0x04·0x08·0x09 = 개행/이름/서식류 |
| **0x0A~0xF6** | **글리프 인덱스(1바이트)** = code − 0x0A  (인덱스 0~236) |
| **0xF7~0xFF (+인자 1바이트)** | **확장 글리프 인덱스(2바이트)** = (code−0xF7)×255 + 0xEC + 인자 |

→ 앞서 "사전 압축"으로 본 0xF7~0xFF는 **사전이 아니라 큰 인덱스용 2바이트 확장**이다.
   메시지 레벨엔 사전/매크로가 없다. 순수 **가변길이 글리프-인덱스 스트림 + 제어코드**.

## 전역 폰트 확인 (핵심)
같은 인덱스가 서로 다른 메시지에서 같은 글자로 나옴 → **전역 폰트 인덱스**:
- こ=0xDD(idx 211), の=0x6D(idx 99), は=0xCC(idx 194), 、=0x7F(idx 117) — 여러 대사서 동일.
- 즉 장면별 폰트가 아니라 **하나의 큰 전역 폰트(≈수백~수천 글리프)**를 인덱싱한다.

## 검증 사례 (라이브 대사)
「エルサリアへようこそ。」(1행 11자) → 스트림 첫 세그먼트가 정확히 **11 글리프**:
`0D 34 14 33 0A (F7 33) D6 82 DD EB 76`
= エ(3) ル(42) サ(10) リ(41) ア(0) へ(287) よ(204) う(120) こ(211) そ(225) 。(108). ✓

## ROM 저장 = 압축
WRAM 디코드 스트림(0x7F6000~)을 ROM에서 검색 → **불일치**. 메시지는 ROM에서 **압축**되어
WRAM 0x7F6000 버퍼로 전개된 뒤 인터프리터가 읽는다. (압축 해제 루틴은 별도.)

## 데이터 흐름 (전체)
```
[ROM: 압축 메시지] --디컴프--> [WRAM 0x7F6000: 글리프-인덱스 스트림]
   --인터프리터(0xAD51)--> 각 인덱스로 [전역 폰트(압축)]에서 글리프 디컴프
   --> VRAM 0xC000 동적 업로드(8×16, 2bpp) --> BG3 표시
```

## 한글패치 파이프라인 (확정 설계)
1. **텍스트 덤프**: 디컴프 루틴 역공학 → ROM에서 전 메시지 스트림 정적 덤프
   (또는 라이브 디버거로 장면별 WRAM 스트림 수집).
2. **인덱스→글자 표**: 전역 폰트를 디컴프/렌더 → 각 인덱스의 실제 글자 확정
   (또는 라이브로 스트림↔화면 대조하여 인덱스맵 구축).
3. **번역**(영문 참조) 후, 한글 글리프를 전역 폰트에 추가 → **새 인덱스**로 스트림 재작성.
4. **재압축**(또는 롬 확장으로 비압축 재배치 + 로더 패치) 및 포인터 갱신.

## 남은 구체 작업
- 메시지 **디컴프 루틴**과 압축 포맷, ROM 메시지 **포인터 테이블** 위치.
- **전역 폰트** 압축 포맷·ROM 위치(글리프 디컴프 `$8ABD/$B18E/$B3D8` 추적).
- 제어코드(0x01~0x09) 의미표(점프테이블 0xADF5 핸들러 분석).
