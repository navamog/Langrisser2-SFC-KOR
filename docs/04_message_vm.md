# 메시지 스크립팅 VM (bank 0x07) — 전체 텍스트 파이프라인

emucap 라이브 디버거로 특정. 텍스트는 단순 압축이 아니라 **다단계 스크립트 VM**이다.

## 전체 파이프라인
```
[ROM: 씬별 메시지 스크립트]  (포인터 $1DD1/$1DD2, 뱅크 0x7F 경유)
      │  VM(bank 0x07) — 명령 해석·매크로 치환
      ▼
[WRAM 0x7F6000: 최종 글리프-인덱스 스트림]  (0x0A~0xF6=인덱스, 0xF7~0xFF=확장, 0x00~09=제어)
      │  인터프리터 0xAD51
      ▼
[전역 폰트(압축)에서 글리프 디컴프] → VRAM 0xC000 동적 업로드(8×16,2bpp) → BG3 표시
```

## VM 진입/루프 (bank 0x07)
- 로더/초기화 `0x07A122~`: `$02/$03` = 소스 포인터(`$1DD1/$1DD2`에서 로드), `$04`=0x7F(소스 뱅크),
  대상 버퍼 `0x7F6000`을 0xFF로 초기화, 대상 포인터 `$1DD6/$1DD7`=0x6000.
- 메인 루프 `0x07A159`:
  ```
  A159 LDA [$02] → $17    ; 소스에서 명령 바이트
  A15D INC $02 (16bit)
  A163 CMP #$FF → JMP $A189 ; 0xFF = 스크립트 종료
  A16A ASL / TAX / JMP ($A1B0,X) ; 그 외 명령 → 점프테이블 분기
  ```
- **명령 점프테이블 `0x07A1B0`** (16개, 워드): 
  `A260 A32C A334 A616 A639 A678 A6CC A722 A762 A7A1 A802 A84E A888 A8B7 A9EE AC6E`

## 명령 0 (`0x07A260`) = 텍스트 복사 + 매크로 치환 (주력)
- 소스 바이트 $17=[$02] 읽어:
  - `0x00` → 서브루틴 복귀 처리(`$A16F`)
  - `0xFD/0xFE/0xFF` → 특수(치환/종료 계열)
  - **≥0xB000** → 이름/변수 참조: `(값−0xB000)`로 **WRAM 테이블 `0x7EB006`** 인덱싱 →
    치환 문자열 포인터(`$17/$19`) 취득. (캐릭터명·수치 등 삽입)
  - 그 외 → 글리프-인덱스로 대상 버퍼(0x7F6000)에 기록.
- 즉 최종 표시 텍스트 = 스크립트의 리터럴 인덱스 + 이름/변수 치환 결과.

## 핵심 주소·변수 요약
| 항목 | 위치 |
|------|------|
| 글리프 인터프리터(디스패치) | ROM 0x00AD51 (미러 0x80) |
| 제어코드 점프테이블 | 0x00ADF5 |
| VM 로더/초기화 | ROM 0x07A122 |
| VM 메인 루프/명령 dispatch | 0x07A159 / 점프테이블 0x07A1B0 |
| VM 명령0(텍스트+치환) | 0x07A263 |
| 이름/변수 치환 테이블 | WRAM 0x7EB006 |
| 메시지 스크립트 소스 포인터 | ZP $02/$03 + 뱅크 $04(=0x7F); 영속본 $1DD1/$1DD2 |
| 최종 글리프-인덱스 버퍼 | WRAM 0x7F6000 |
| 글리프 렌더 서브루틴 | 0x008ABD / 0x00B18E / 0x00B3D8 |

## write BP 주의(emucap)
- WRAM 대상 write 브레이크는 **memory_type=snesMemory + CPU주소(0x7F6000)** 로 걸어야 잡힘.
  `snesWorkRam` 오프셋으로는 이 VM/블록무브 쓰기를 못 잡았다.
- press_buttons 중엔 write BP가 안 멈추는 듯 — **run_frames 중에 히트**해야 interrupted 반환.

## ROM 스크립트 원본 (미확정 — 추적 필요)
- 실행 중 VM 소스 `$1DD1/$1DD2 = 0xC03B`(+뱅크 0x7F) → 스크립트 WRAM 사본 **0x7FC000~**.
  WRAM 스크립트 앞부분: `0C 00 0E 00 10 00 12 00 14 00 16 00 FF FF… 18 00 48 00 38 04 0A 4C 3B 00
  02 00 F6 00 00 01 02 00 F6 00 00 02 …`(`02 00 F6 00 00 0X` 반복 = 어떤 명령+인덱스).
- ⚠ ROM 0x034DF6은 접두사만 우연 일치 — 오프셋을 따라가면 **65816 코드**(텍스트 아님). 무관.
- 따라서 WRAM 0x7FC000 스크립트의 **ROM 출처는 아직 미발견**. 다음으로 이 WRAM 영역에
  snesMemory write BP(0x7FC000)를 걸어 **ROM→WRAM 적재 루틴/소스**를 잡아야 함.
- WRAM 사본은 실행 중 in-place 수정됨(포인터 갱신).

## 한글패치 관점 남은 작업
1. **메시지 스크립트 소스**: $1DD1/$1DD2가 가리키는 실제 위치(0x7Fxxxx는 WRAM이므로,
   그 이전에 ROM→WRAM으로 스크립트를 적재하는 단계가 있음) → ROM상 원본·포인터테이블 특정.
2. VM 명령 16종 의미 정리(특히 리터럴 텍스트/치환) → 정적 덤프·재작성 도구.
3. **전역 폰트** 압축포맷·위치 → 인덱스↔글자 전역표.
4. 재삽입: 스크립트 리터럴 텍스트를 한글 인덱스로 교체 + 폰트에 한글 글리프 추가.
